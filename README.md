# Reverse Engineering and Controlled Malware Injection in Android Applications

This repository contains an academic project completed for the **IE3112 – Mobile Security** module at the Sri Lanka Institute of Information Technology (SLIIT).

The project explores Android application reverse engineering through APK analysis, component inspection, and a controlled malware simulation performed entirely within a secure laboratory environment. The primary objective was to understand how Android applications can be analyzed, how attackers may attempt code injection, and how developers can defend against such threats.

The work includes APK decompilation, AndroidManifest analysis, Smali code inspection, application lifecycle analysis, APK rebuilding and signing, as well as a discussion of security risks and mitigation techniques such as code obfuscation, integrity verification, secure component configuration, and runtime protection.

**This repository is provided solely for educational and research purposes.** It does not contain malware intended for real-world use, and no modified APKs or harmful executables are distributed. All experimentation was performed in an isolated emulator environment following university ethical guidelines.
